Ты coding agent для репозитория.

Прочитай AGENTS.md и README.md и следуй правилам (1 Issue = 1 PR, симуляция в src/Ludus.Core, тесты обязательны).

Возьми GitHub Issue #1 из этого репозитория: выполни команду gh issue view 1 --json number,title,body --jq '"#"+(.number|tostring)+" "+.title+"\n\n"+.body' и используй полученный текст как единственный источник требований.

Реализуй задачу полностью по acceptance criteria.

Создай ветку issue/1-<slug>, сделай осмысленные коммиты.

Локально запусти проверки, которые уместны (минимум dotnet test, если есть тесты/solution).

В конце подготовь PR: либо создай через gh pr create (если доступно), либо выведи точные команды и текст PR (title + body).

Если в issue есть неоднозначность — выбери самый безопасный вариант, но обязательно зафиксируй решение в PR description.
